## Detalhe dos erros

### 1 — SyntaxError na projeção (v4.32.0)
Ao inserir `calcProjecaoMes_` / `drawProjecaoChart_`, o cabeçalho de `drawLineChart` foi **substituído** em vez de inserir **antes**. Corpo órfão + `}` sobrando → JS não parseava → `goTo`/`init` mortos.

### 2 — HTML no lugar de JS (v4.32.1)
Fallback de rede do SW caía em `index.html` para `zc-*.js`. Cache envenenado.

### 3 — Banner full-screen de reparo (v4.32.4)
Só sumia se `init()` rodasse. Com JS morto, loop de “ZapClin precisa reparar o cache”.

### 4 — Clear sem reload
Com `?repaired=1`, limpava caches **depois** dos scripts já carregados e **não** recarregava.

### 5 — `zc-version.js` incompleto
Reescrito só com `APP_VERSION`, sem `WEB_APP` / `BACKEND_MIN_VERSION` / demais constantes.

### 6 — Restauro para commit já quebrado
“Última válida” não era 4.32.0. Última com JS válido: **v4.31.1** (`a532fab`).

### 7 — Sem gate de sintaxe
Publicou-se várias vezes sem `node --check` no script que contém `goTo`/`init`.

### 8 — Hotfix de SW sem causa raiz
Tratou cache enquanto o parse error seguia no `index.html`.

### 9 — Canvas invisível
Desenho dependia de `offsetWidth` no paint; vinha 0 → gráfico “inexistente”.

### 10 — SW off + boot forçado
Removeu o toast “Nova versão disponível…” (padrão ZapClin) e prendeu status em versões antigas.

### 11 — Letterbox no SVG
`viewBox` fixo 640px + `preserveAspectRatio=meet` → plot estreito no centro do card.

### 12 — Fonte Syne em valores
KPIs de projeção usaram `--font-head` (Syne). Padrão de dinheiro/dados: **Inter** (`--font-data`).

### 13 — Bloco sob o gráfico sem pedido
Texto “Método…” + tabela DIA/REAL/RITMO/DESVIO. Removido em v4.33.0.

### 14 — Despadronização do fluxo de versão
Boot nuclear a cada bump conflitou com `REGRAS_DE_PUBLICACAO_SEGURA` (`?force=`, SW, toast).

## Proibições absolutas (não repetir)

1. Nunca publicar `index.html` sem `node --check` no script inline com `goTo`/`init` + balanceamento de `{`/`}`.
2. Inserir funções novas **antes** de funções existentes — nunca trocar a assinatura deixando o corpo para trás.
3. SW: **nunca** servir HTML como fallback de `.js` / `.css` / `manifest`.
4. Sem UI full-screen de “reparo” que dependa de JS para desaparecer.
5. Após limpar SW/cache: **sempre** `location.replace` limpo (fluxo oficial: `?force=vX.Y.Z`).
6. Não editar `zc-version.js` sem preservar **todas** as constantes (`WEB_APP`, `BACKEND_MIN_VERSION`, …).
7. Em restauro: último commit com **JS válido**, não a última feature.
8. Não desligar o Service Worker “para recovery” sem restaurá-lo na mesma entrega.
9. Não inventar banner permanente de versão — ZapClin usa **toast** do SW + `aplicarControleVersao_`.
10. Tipografia de valores monetários: **Inter** (`--font-data`), não Syne.
11. SVG de dashboard: medir largura do card; evitar letterbox (`viewBox` alinhado ao CSS).
12. Não empilhar UI sob o gráfico sem pedido explícito do sócio.
13. Escopo FE = FE. Não redesenhar fluxo de update/cache no meio de feature visual.
14. Informar URL `?force=vX.Y.Z` após cada deploy FE (já na Regra 4).
