# Erros do agente — PWA ZapClin (14/07/2026)

Registro explícito dos erros que derrubaram o frontend e das regras para não repetir.

## Erros

1. **SyntaxError na v4.32.0 (projeção)**  
   Ao inserir `calcProjecaoMes_` / `drawProjecaoChart_`, o cabeçalho de `drawLineChart` foi **substituído** em vez de inserir **antes**. O corpo de `drawLineChart` ficou órfão + `}` sobrando. Resultado: JS não parseava → `goTo`/`init` nunca rodavam → Home “viva” em HTML, navegação morta, status preso em `v4.7`.

2. **Service Worker devolvendo HTML no lugar de JS (v4.32.1)**  
   Fallback de rede caía em `index.html` para pedidos de `zc-*.js`. Cache envenenado; app quebrava em qualquer cliente com SW antigo.

3. **Banner full-screen de “reparar” (v4.32.4)**  
   O banner só sumia se `init()` rodasse. Com JS morto, a tela fica presa no reparo em loop. Piorou a percepção de “sistema destruído”.

4. **Bootstrap que limpa cache e não recarrega**  
   Com `?repaired=1` na URL, limpava SW/caches **depois** dos scripts já terem carregado do veneno e **não** fazia `location.replace`. Cliente ficava no mesmo documento quebrado.

5. **Sobrescrever `zc-version.js` incompleto**  
   Em um hotfix, o arquivo foi reescrito só com `APP_VERSION`, sem `WEB_APP` e demais constantes — teria matado a API se tivesse ido ao ar assim.

6. **Restauro “cego” para v4.32.0**  
   A “última válida” não era 4.32.0: ela já tinha o SyntaxError. A última com JS válido era **v4.31.1** (`a532fab`).

7. **Hotfixes em cascata sem validar sintaxe do app inline**  
   Várias publicações em `main` sem `node --check` no script principal. O bug estrutural ficou vivo enquanto se “consertava” SW/cache.

8. **Mudança agressiva de SW (network-first / novo arquivo / stub autodestroi)** enquanto o JS base já estava inválido  
   Tratou sintoma (cache) e mascarou a causa (parse error).

## Regras daqui pra frente

- Nunca publicar `index.html` sem validar o script inline que contém `goTo`/`init` (`node --check`, balanceamento de chaves).
- Inserir funções novas **antes** de funções existentes; nunca “trocar a assinatura” deixando o corpo para trás.
- SW: **nunca** servir HTML como fallback de `.js`/`.css`.
- Sem banner full-screen que dependa de JS para sumir.
- After clear de SW/cache: **sempre** um reload limpo.
- Não editar `zc-version.js` sem preservar `WEB_APP`, `BACKEND_MIN_VERSION`, etc.
- Em “restaurar última válida”: escolher o último commit com **JS válido**, não só a última feature.

## Estado após correção

- Base estável: conteúdo da **v4.31.1**
- Feature de projeção republicada com JS validado: **v4.32.7**
- SW permanece desativado nesta recovery até estabilizar

## Erro adicional (gráficos “sumidos”)

9. **Canvas da projeção invisível (v4.32.7)**  
   O bloco existia no HTML, mas o desenho em `<canvas>` dependia de `offsetWidth` no momento do paint. Em alguns carregamentos a largura vinha 0 → gráfico “não existia” na tela.  
   **Correção v4.32.8:** projeção em **SVG/DOM** com `viewBox` fixo + área CSS de 220px; redesenho dos charts com `rAF` + try/catch.
